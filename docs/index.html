MetaSerialization

meta-serialization is a framework for simplifying writing new serialisation libraries for the swift standard library environment (Encodable, Decodable, etc.)
It’s aim is to let anyone create a serialization library that works with the swift serialization framework by nearly just writing the real serialization code.

current status

This framework has a incomplete code-base and is not tested execpt for one simple automated testing case. Documentation will be extended.

What it does

meta-serialization provides a layer inbetween your serialization framework and the swift standard library interface (mainly Encoder and Decoder) and furthermore provides you blueprints for some kinds of frontends.

What it does in many words

Terminology if you’re new to serialization (will not replace wikipedia research)

Serialization is converting swift objects to and from a transferable or storable format.
Once again: When you wan’t to convert any object in your code into JSON (or Plist or XML or whatever) you need serialization.

Quick introduction to the problem of writing serialization librarys

In the first place, making a class or struct or enum decodable (external format (e.g. JSON) to swift) or encodable (swift to external format) is relatively easy, but at least there are many articles online, where you will find out how to do this and the swift documentation will also tell you, but as soon as you need your own serialization library, that does the transition between swift and external, help get’s realy rare and the only reference you get (or at least was found by me) is the source code of JSONEncoder in the swift library. Also the swift standard framework is a little bit strange in the first place and nearly weird when you need to write transition code, but a lot of the code is duplicated every time you adopt a new format (JSONEncoder and PlistEncoder in swift/stlib on github look nearly the same). Here meta-serialization sets of.

This frameworks work

meta-serialization provides a MetaEncoder and MetaDecoder, that depend on a so-called Translator.

MetaDecoder and MetaEncoder create a representation of the serialized data in a meta format. This format is created in a way, that in the end the meta tree (as I call the in-between format) will only contain values, that are natively supported by your framework or in other words that you can encode directly. This enables you to serialize any swift class or struct or enum you write, plus any array, dictionary, tuple, Int, String, and so on to an external (I call it raw) representation that only supports, say, Numbers, String and some kind of Dictionary by just writing the code to convert a swift Int to your representation of a number, a swift String to your representation of a String and Dictionary of exclusively your supported types (Number, String, Dictionary in this example) to your representation of a keyed container or Dictionary (or Map).

Installation

Currently only carthage is supported (swift package manager is only half-way supported). I can not guarante, that cartage will work out, It builds on my computer and all schemes are set public, but I never tried installing it somewere else. I did not manage to add full swift-package-manager support or cocoapods support.

Carthage

Add the following line to your projects Cartfile:

github "cherrywoods/swift-meta-serialization"

For more information about cartage and it’s usage, please consult carthage’s github repository.

Swift Package Manager (iOS only)

You may use the swift package manager, but only for the iOS framework. The other three targets aren’t supported because refering to the swift package manager repository It is an error if the paths of two targets overlap (unless resolved with exclude) (If I missunderstood that part please open a pull request). So, if you need the macOS, tvOS or watchOS target, you may not (simply) use the swift package manager, but the code for all the targets is pretty much the same (there is a little additional code for macOS, but these code only includes a String wrapping Meta For Float80). However, insert the following code into your Package.swift file, into the dependencies array of your Package.

.package(url: "https://github.com/cherrywoods/swift-meta-serialization.git", from: "0.0.4"),

For more information about the swift package manager, visit https://swift.org/package-manager/.
