Protocols

The following protocols are available globally.

  • EncodingRepresentation

    A protocol for encoded data, that will only be written.

    Please refer to the documentation of Representation for more information about usage.

    However, this protocol gives you also more control about the encoding process, because you will have to instantiate a new MetaEncoder, that you may pack with more information, than TranslatingCoder can handle. For example you may provide a userInfo dictionary or an initial codingPath.

  • GenericMeta

    A subprotocol of Meta, enabeling you to rely on a specfic type, so you don’t need to check for this type all the time

    meta-serialization provides default implementations for set(value:) and get(). set(value:) will crash (using the precodingtion method) the programm if a value with the wrong type is passed to set(value:). However, meta-serialization will never do this, unless you do not implement wrappingMeta(forSwiftType:) in your implementation of Translator with faults.

  • UnkeyedContainerMeta

    A meta for a unkeyed collection of metas.

    The indexiation starts at 0 and ends at count-1, with steps of size 1 (if you have 4 elements, they have to be indexed 0,1,2,3).

    You may not use custom indexes, if you liked meta-serialization to work properly.
